## OS-Visible Paths

macOS Android Bluetooth HID and Windows VHF are equal primary v1 OS-visible paths. Android Bluetooth HID gives macOS Apple Silicon a normal gamepad-style joystick without Apple virtual HID entitlements. Windows VHF gives Windows 11 x64 a normal gamepad-style virtual joystick with output-to-phone-haptic routing.

Do not treat Windows VHF as a barely-mentioned fallback. Use the Android HID path for macOS input and the Windows VHF path for Windows input; use the LAN visualizer path for diagnostics, replay, pairing/control proof, and phone haptic troubleshooting.

## Unsupported or Deferred in v1

These rows are explicit v1 known limits, not hidden setup gaps. Required deferred/unsupported phrases are direct desktop-to-gun Bluetooth, physical gun motor rumble, and game-specific presets.

| Row | v1 status |
|-----|-----------|
| Direct desktop-to-gun Bluetooth | Unsupported. Android remains gun host and motion source. |
| Physical gun motor rumble | Deferred. v1 feedback is Android phone vibration. |
| macOS HID haptics through Android Bluetooth HID | Unsupported/deferred for the stable descriptor. Use LAN or Windows VHF phone-haptic paths. |
| Game-specific presets | Deferred. v1 validates the Default Visualizer profile and profile editor behavior, not named game presets. |

Use [v1 compatibility limits](limits/v1-compatibility-limits.md) for evidence pointers and next proof required to change any unsupported, fallback, or deferred row.
