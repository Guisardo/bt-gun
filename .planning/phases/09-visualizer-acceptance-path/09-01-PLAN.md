## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ControlServer -> UI event hub | Authenticated callbacks cross into multiple UI/backend consumers. |
| Accepted UDP input -> metrics/model | Trusted accepted frame fields become user-visible latency/loss and controls. |
| Manual checklist state | Local user confirmation can affect pass state but must not affect transport trust. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | `DesktopUiEventHub` | mitigate | Fanout tests prove callbacks are preserved and all consumers receive trusted events. |
| T-09-02 | Spoofing | `VisualizerModel` checklist rows | mitigate | Separate observed from confirmed states; physical and OS-visible proof rows require explicit confirmation. |
| T-09-03 | Tampering | `VisualizerMetrics` packet loss | mitigate | Accept only trusted `UdpReceivedInput` sequence gaps and reset on session change. |
| T-09-04 | Information Disclosure | model labels/diagnostics | mitigate | Tests reject secret-like labels and keep raw debug collapsed by default. |
| T-09-SC | Tampering | package installs | accept | No new npm/pip/cargo/Gradle dependencies are installed in this plan; existing repo dependencies only. |
</threat_model>
