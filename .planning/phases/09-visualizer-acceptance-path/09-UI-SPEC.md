# Phase 09 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Boundary

Phase 9 delivers the user-visible desktop Swing visualizer that proves the v1 path before any commercial game support.

The visualizer must prove the authenticated LAN visualizer path, macOS Android Bluetooth HID input, Windows VHF input, recenter behavior, latency, packet loss, LAN phone haptics, and Windows output-to-phone haptics. macOS Bluetooth HID output/haptics may pass as unsupported/deferred only when the row shows current evidence for the known limitation.

Out of scope for this UI contract: game-specific presets, generated evidence bundles as the primary pass artifact, physical gun motor rumble, import/export, replay tooling, Phase 10 docs/replay systems, direct desktop-to-gun Bluetooth, desktop profile editing, and a web or Compose replacement for the Swing desktop companion.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Java Swing for desktop companion |
| Icon library | none |
| Font | Swing system Dialog |

No `components.json`, Tailwind, React, Next.js, Vite, or web component system is present. The shadcn gate is not applicable.

Use the existing desktop companion style: utilitarian Swing controls, titled diagnostic panels, compact rows, explicit labels, fixed-format visual surfaces, and no decorative cards. The visualizer is an acceptance harness, not a dense debug dashboard.

Primary scan order:

1. Guided acceptance checklist progress.
2. Live gamepad panel.
3. Latency, packet loss, and stream health.
4. Recenter, aim-zero, and recent event strip.
5. Haptic proof controls and per-platform proof rows.
6. Collapsed raw debug drawer.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline value gaps, tick labels, small badge padding |
| sm | 8px | Button gaps, checklist row inner gaps, event-strip item gaps |
| md | 16px | Panel inner padding, row groups, crosshair label offset |
| lg | 24px | BorderLayout gaps, major horizontal split between checklist and gamepad |
| xl | 32px | Window outer border and top-level section padding |
| 2xl | 48px | Minimum command button height and large status band height |
| 3xl | 64px | Reserved for full-window breakpoints only |

Exceptions: live stick and aim surfaces use fixed square sizes between `180px` and `240px`; button indicators use fixed `44px` minimum hit/scan targets; the event strip shows exactly 10 fixed-height event slots; the raw debug drawer may use a scroll pane after `320px` height.

## Typography

Use exactly these sizes and weights for new Phase 9 UI. Do not add more type sizes unless an existing Swing look-and-feel widget imposes one.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 13pt | 400 | 1.4 |
| Body | 14pt | 400 | 1.5 |
| Heading | 16pt | 700 | 1.2 |
| Display | 22pt | 700 | 1.2 |

Body copy is for checklist row text, metrics labels, event-strip labels, haptic result text, and diagnostic values. Label is for numeric values such as latency, loss, sequence, axes, and raw debug fields. Heading is for visualizer section titles. Display is reserved for the window title `BT Gun Visualizer` and the top pass/fail summary only.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F4F7F8` | Window background and quiet default surface |
| Secondary (30%) | `#FFFFFF` | Checklist panel, gamepad panel, metrics panel, debug drawer |
| Accent (10%) | `#0C5D80` | Primary haptic test action, active checklist row, live aim dot, active stream badge, focus ring |
| Destructive | `#B02A37` | Failed proof row, failed haptic result, destructive reset confirmation only |

Accent reserved for: `Run phone haptic test`, active checklist row, live mapped aim dot, active stream badge, selected confirmation row, and keyboard focus ring. Do not apply accent to every button or every live value.

Supporting colors: `#181F29` for primary text, `#596573` for muted text, `#DCE4E8` for borders, `#BAC7CF` for strong borders, `#E0F3F8` for non-destructive active tonal backgrounds, `#14765C` for pass states, `#B35000` for stale/degraded warnings, and `#FFEFF0` for failed-state background.

## Visual Layout Contract

### Window Structure

Add a separate `VisualizerWindow`; do not turn `PairingWindow` into the main visualizer.

Default desktop layout:

| Region | Content | Behavior |
|--------|---------|----------|
| North | `BT Gun Visualizer`, session summary, active Android profile summary | Always visible |
| Center left | Guided acceptance checklist | Always visible; rows update from live state and user confirmation |
| Center right | Live gamepad panel with controls, stick, aim, stale overlay | Always visible; stale/disconnected overlay never destroys last context |
| South | Latency/loss metrics, haptic proof controls, recent event strip | Always visible unless window too small |
| Collapsible drawer | Raw provider, yaw, pitch, roll, raw aim, rejection reasons | Collapsed by default; enabled when Android raw debug is on |

The top of the visualizer must show checklist and live gamepad together. The first viewport must never be only diagnostics.

### Fixed Visual Surfaces

| Surface | Contract |
|---------|----------|
| Button indicators | Six fixed indicators: Trigger, Reload, X, Y, A, B. On = filled pass/active color; off = bordered neutral; stale = warning overlay. |
| Stick crosshair | Square canvas; center axes visible; dot clamps to edge; neutral center visible. |
| Aim crosshair | Square canvas; mapped aim dot uses accent; zero marker visible; stale overlay preserves last aim. |
| Stale overlay | Semi-transparent warning band reading `stale` or `disconnected`; never clears the last accepted aim point. |
| Event strip | Last 10 product events, newest first or leftmost; include event type, sequence, and age. |

### Checklist Rows

Required rows:

| Row ID | Label | Pass Source |
|--------|-------|-------------|
| `lan_visualizer_stream` | LAN visualizer stream | Observed authenticated session plus accepted mapped UDP frame |
| `live_controls` | Live gun controls | Observed trigger, reload, joystick, X/Y/A/B, stick, and mapped aim activity |
| `recenter_aim_zero` | Recenter and aim zero | Observed recenter/aim-zero status plus user confirmation |
| `macos_hid_input` | macOS Android HID input | User confirmation after OS-visible macOS input proof |
| `windows_vhf_input` | Windows VHF input | Backend diagnostics plus user confirmation after Windows target proof |
| `lan_phone_haptic` | LAN phone haptic | Haptic command ack plus user confirmation of phone vibration |
| `windows_vhf_haptic` | Windows VHF phone haptic | Windows backend routed output plus user confirmation |
| `macos_hid_haptic_limit` | macOS HID haptic limitation | Unsupported/deferred evidence displayed and user confirmation |
| `latency_target` | Latency under 50 ms | Current-session headline latency sample under target during normal local Wi-Fi |
| `packet_loss` | Packet loss visible | Expected/missed/percent counters visible for current stream |

Checklist states: `waiting`, `observed`, `confirmed`, `failed`, `unsupported/deferred`. Rows requiring physical feel, OS visibility, or phone vibration must include user confirmation; observed state alone is not sufficient.

## Interaction Contract

### Open and Lifecycle

| Event | Required UI Behavior |
|-------|----------------------|
| Android authenticated | Auto-open `VisualizerWindow` if not already open |
| PairingWindow action | Show visible `Open visualizer` button |
| LAN disconnect | Keep visualizer open, show stale/disconnected state, preserve checklist progress |
| Reconnect | Resume live updates in same window and show current session status |
| Window close | Hide or dispose only visualizer UI; do not stop control server or backend runtimes |

### User Actions

| Action | Copy | Result |
|--------|------|--------|
| Start LAN phone haptic | Run phone haptic test | Sends authenticated `HapticCommand`, then shows queued/ack/fail |
| Confirm row | Confirm observed | Marks physical/OS-visible proof row confirmed |
| Mark unsupported row | Confirm limitation | Marks macOS HID haptic limitation as unsupported/deferred with evidence visible |
| Reset checklist | Reset checklist | Requires destructive confirmation copy from this contract |
| Toggle debug drawer | Show raw debug / Hide raw debug | Expands or collapses secondary raw data only |

Do not add profile editing, mapping controls, stream secret display, pairing material display, raw logs, device identifiers, screenshots, or evidence export in Phase 9.

## Metrics Contract

### Latency

Headline label: `Latency: {ms} ms | target <50 ms`.

The headline metric is Android capture timestamp to Swing visualizer update/render. Because Android `elapsedRealtimeNanos` and JVM `System.nanoTime()` have unrelated origins, the UI must label this as an offset-estimated value unless a validated time-sync estimate exists.

Required fields:

| Field | Display |
|-------|---------|
| Headline latency | `Latency: {ms} ms | target <50 ms | {pass/warn}` |
| Offset quality | `Clock offset: {good/estimated/unavailable}` |
| Capture-to-send | `Capture to send: {ms} ms` |
| Receive-to-render | `Receive to render: {ms} ms` |
| Sample age | `Last input: {ms} ms ago` |

Direct subtraction of Android capture timestamp from desktop render timestamp is forbidden unless converted through an explicit offset estimate.

### Packet Loss

Headline label: `Packet loss: {missed}/{expected} ({percent}%)`.

Packet loss is current-session expected/missed count from accepted UDP sequence gaps only. Reset counters on new stream session or control session. Rejected duplicate, old, unauthenticated, malformed, or stale frames may appear in diagnostics but must not count as missed accepted-session packets.

## Haptic Proof Contract

| Proof | UI Requirement |
|-------|----------------|
| LAN phone haptic | `Run phone haptic test` sends authenticated LAN command and shows queued, ack, fail, or no active session. |
| Windows VHF phone haptic | Guided row instructs user to trigger Windows output/rumble path; row passes only after backend routed output and user confirms phone vibration. |
| macOS HID haptic limitation | Row displays unsupported/deferred current evidence; can pass as `unsupported/deferred` only after user confirmation. |

Haptic result copy:

| State | Copy |
|-------|------|
| Queued | Phone haptic queued |
| Ack | Phone haptic confirmed |
| Failed | Phone haptic failed. Check Android session and try again. |
| No session | No active Android session. Pair Android before running haptic proof. |
| Unsupported | macOS HID haptic unsupported on current path; LAN and Windows phone haptics remain available. |

## Recenter and Debug Contract

Recenter display:

| Row | Required Format |
|-----|-----------------|
| Current aim zero | `Aim zero: {ready/recentered/pending/unavailable}` |
| Recenter control | `Recenter: hold reload for 2000ms` |
| Last recenter | `Last recenter: {age} ago` or `Last recenter: none` |

Raw debug drawer:

| Field | Required Format |
|-------|-----------------|
| Provider | `Provider: {id}` |
| Yaw/pitch/roll | `Yaw: {value} | Pitch: {value} | Roll: {value}` |
| Raw aim | `Raw aim: x={value} y={value}` |
| Raw debug state | `Raw debug: on` or `Raw debug: off` |
| Rejection reason | `Last rejection: {reason}` with secret redaction applied |

Raw debug data is secondary and must never crowd the default live gamepad panel.
