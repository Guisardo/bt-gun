## Phase Goal

**As a** macOS desktop user, **I want to** use the live Android gun stream as a regular macOS gamepad-style joystick with honest output capability reporting, **so that** I can validate v1 input and phone haptics before later desktop UX work.

<objective>
Implement the macOS backend bridge over a thin native helper boundary.

Purpose: Connect tested Kotlin report bytes to a CoreHID helper without moving LAN/session/security/desktop mapping/haptic transport responsibilities into native code.
Output: Tested helper client, macOS virtual controller backend, honest capability preset, and helper line protocol.
</objective>

<interfaces>
Helper line protocol:
- Kotlin sends `HELLO 1`, `SUBMIT_INPUT &lt;hex&gt;`, `READ_OUTPUT`, `STATUS`, and `QUIT`.
- Helper returns `OK`, `ERR &lt;safe-token&gt;`, `OUTPUT &lt;hex&gt;`, or `STATUS {"version":1,"deviceActive":bool,"osVisible":bool,"setReportCallbackSeen":bool,"inputReportsSubmitted":long,"outputReportsQueued":long,"malformedInputReports":long,"malformedOutputReports":long}`.
- Helper stderr is discarded or sanitized; helper never sees QR data, pairing secrets, stream keys, HMAC keys, session ids, or `HapticCommand`.
</interfaces>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Kotlin backend -> helper process | Kotlin sends report bytes to native helper over local stdin/stdout. |
| Helper process -> Kotlin backend | Helper returns status and output report bytes. |
| Helper process -> macOS HID stack | Helper dispatches reports to CoreHID. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-09 | Spoofing | Helper process | mitigate | Kotlin launches explicit helper command, requires `HELLO 1`/status version match, validates command entries nonblank, and fails closed on unexpected responses. |
| T-07-10 | Tampering | Helper input/output frames | mitigate | Validate hex, report IDs, exact lengths, reserved output bytes, and status JSON before publish or haptic mapping. |
| T-07-11 | Information Disclosure | Helper boundary | mitigate | Helper protocol excludes LAN/session IDs, QR/manual codes, proof values, stream keys, HMAC keys, private keys, and raw UDP payloads. |
| T-07-12 | Repudiation | Simulated output reports | mitigate | Keep `simulateOutputReport` test-only; capability/proof docs state D-07/D-08 require OS-origin output in later plans. |
| T-07-SC | Tampering | Package installs | accept | No npm, pip, cargo, or new Gradle dependency install in this plan. |
</threat_model>
