<objective>
Define the Android Bluetooth HID descriptor and deterministic input report packer.

Purpose: macOS can only see the Android phone as a controller if Android registers a regular gamepad-style HID shape and sends stable report bytes from live gun/motion state.

Output: Android HID descriptor, report packer, and golden tests for ANDR-10.
</objective>

## Phase Goal

**As a** user with the discontinued iPega AR gun, **I want to** use the Android phone as a Bluetooth HID gamepad for macOS Apple Silicon, **so that** macOS sees a normal controller without paid Apple virtual HID entitlements while the completed Windows VHF path remains fallback.

## Artifacts this phase produces

- HID descriptor constants for report ID `0x01` input and report ID `0x02` output.
- Input report bytes for trigger, reload, X/Y/A/B, stickX/stickY, and aimX/aimY.
- Golden tests proving parity with the locked BT Gun descriptor contract.
