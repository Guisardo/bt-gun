- Service-owned HID lifecycle tied to the visible foreground session boundary.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User tap -> foreground service action | HID advertising/registering begins only after explicit action. |
| Live gun/motion state -> Bluetooth HID host | Input reports leave Android over Bluetooth. |
| HID output -> phone haptic executor | Host-origin bytes can trigger vibration only after validation. |

| T-07-13 | Tampering | Service state fanout | mitigate | Send reports only through tested `AndroidBluetoothHidGamepad` gate. |
| T-07-14 | Information Disclosure | Dashboard | mitigate | Status strings avoid host Bluetooth addresses and personal phone identity. |
| T-07-15 | Elevation of Privilege | HID lifecycle | mitigate | Stop/unregister on explicit stop, session stop, service destroy, and virtual cable unplug. |
