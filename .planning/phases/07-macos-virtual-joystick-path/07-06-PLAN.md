<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CoreHID gate -> DriverKit fallback | Failed user-space proof triggers system-extension work. |
| Host app -> DriverKit provider | App/user-client sends HID report bytes into a privileged extension boundary. |
| System Extension approval -> OS state | User approval changes macOS extension state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-22 | Elevation of Privilege | DriverKit user client/provider | mitigate | Scope provider to HID report bytes, require explicit entitlements, document user-client access, and avoid LAN/session/security code in extension. |
| T-07-23 | Tampering | DriverKit output reports | mitigate | Validate report ID/version/length/reserved bytes in Kotlin mapper; provider forwards bytes only. |
| T-07-24 | Spoofing | Fallback trigger | mitigate | Require Plan 07-05 manifest gate before creating/running fallback; ambiguous gate blocks. |
| T-07-25 | Information Disclosure | System extension logs/evidence | mitigate | Redact signing identities, provisioning details, private keys, device ids, raw logs, QR/proof/session data, and screenshots from committed artifacts. |
| T-07-26 | Denial of Service | System extension activation/rollback | mitigate | Blocking user approval, documented rollback commands, and no install if entitlement unavailable. |
| T-07-SC | Tampering | Package installs | accept | No npm, pip, cargo, or new dependency install; Xcode/Apple entitlements are human setup if required. |
</threat_model>
