<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Trusted UDP callback -> backend publish | Authenticated stream data becomes macOS HID input reports. |
| Helper output reports -> ControlServer haptic send | macOS-origin output bytes can trigger Android phone haptics through authenticated control. |
| Launch flags -> helper execution | Local config selects native helper process path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-13 | Spoofing | Helper launch path | mitigate | Require explicit `btgun.macos.hid.helper.path`, nonblank command, helper handshake/status, and fail closed on missing path. |
| T-07-14 | Tampering | Runtime output haptics | mitigate | Drain only backend output bytes, validate through `MacosOutputReportMapper`, preserve haptic TTL, and send through authenticated `ControlServer`. |
| T-07-15 | Information Disclosure | Runtime diagnostics/UI | mitigate | Display lifecycle/result/stale/source-sequence/haptic status only; no raw packets, secrets, proof values, stream keys, private keys, device ids, or screenshots. |
| T-07-16 | Repudiation | OS-origin proof | mitigate | Runtime tests do not count as final D-07/D-08 proof; later plans require CoreHID output probe and live user confirmation. |
| T-07-SC | Tampering | Package installs | accept | No npm, pip, cargo, Homebrew, or new Gradle dependency install in this plan. |
</threat_model>
