## Phase Goal

**As a** macOS desktop user, **I want to** use the live Android gun stream as a regular macOS gamepad-style joystick with honest output capability reporting, **so that** I can validate v1 input and phone haptics before later desktop UX work.

<objective>
Create pure Kotlin macOS HID report packing and output report validation.

Purpose: Lock deterministic report bytes before native helper, backend lifecycle, and live runtime wiring consume them.
Output: TDD-covered Kotlin packer and output mapper mirroring the accepted Windows runtime shape without moving session security into native code.
</objective>
