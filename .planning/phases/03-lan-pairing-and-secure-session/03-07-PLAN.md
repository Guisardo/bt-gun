## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Android QR scanner to control client | Camera scan result becomes a network endpoint and proof input. |
| Trusted desktop store to reconnect action | Stored metadata is offered to the user for explicit reconnect. |
| Android service lifecycle to network socket | Foreground service controls whether socket is active. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | Android trusted-desktop reconnect | mitigate | Explicit user action plus fingerprint validation; mismatch moves to trust-problem state. |
| T-03-02 | Spoofing | manual fallback | mitigate | Manual code uses proof verifier and rate-limit state from Plan 03-03. |
| T-03-04 | Elevation of Privilege | UI-to-client lifecycle | mitigate | `HostSessionService` owns socket lifecycle and stops it with the foreground session. |
| T-03-05 | Denial of Service | QR/manual inputs | mitigate | Parser and client tests reject malformed/expired/unreachable state and expose rescan/manual-edit actions. |
| T-03-06 | Information Disclosure | Android UI | mitigate | UI displays state and fingerprint suffix only; no QR secret/proof/private key in diagnostics. |
</threat_model>
