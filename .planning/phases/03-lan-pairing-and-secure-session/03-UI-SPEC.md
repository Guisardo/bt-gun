## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline separators, QR metadata gaps |
| sm | 8px | Field vertical padding, compact form gaps |
| md | 16px | Default Android screen padding and grouped control spacing |
| lg | 24px | Pairing section gaps and desktop QR/manual fallback separation |
| xl | 32px | Desktop panel outer padding and major Android action gaps |
| 2xl | 48px | Minimum button/touch target height |
| 3xl | 64px | Large desktop window top/bottom breathing room only |

Exceptions: Android values are `dp` and text values are `sp`; desktop values are pixels. QR image must be at least 240x240. Pairing-code cells must be 40x48 minimum. Long fingerprint text must wrap instead of shrinking below 13sp/13px.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 13px/sp | 400 | 1.4 |
| Body | 14px/sp | 400 | 1.5 |
| Heading | 20px/sp | 700 | 1.2 |
| Display | 22px/sp | 700 | 1.2 |

Use only these four sizes. `Display` is reserved for `BT Gun Host` and desktop companion window title. Field labels, diagnostics, endpoint text, heartbeat age, and fingerprint text use `Body` or `Label`, never heading scale.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F7F8F6` | Android screen background and desktop window background |
| Secondary (30%) | `#FFFFFF` | QR panel, manual entry panel, trusted-desktop list rows, diagnostics panel |
| Accent (10%) | `#14765C` | Connected state, successful pairing state, primary action focus/selection, verified trust indicator |
| Destructive | `#B3261E` | Forget trusted desktop, trust mismatch warning confirmation, unrecoverable security error |

Accent reserved for: `Connected`, `Pairing ready`, `Trusted`, active primary CTA state, and QR scan success. Do not use accent for every clickable element.

Additional state colors: text `#1F2933`; secondary text/borders `#4E5D6C`; manual/debug info `#345BB2`; degraded/expiry warning `#B35000`; inactive/pending `#4E5D6C`.

## Interaction Contract

### Desktop Companion Pairing Surface

| Region | Contract |
|--------|----------|
| Header | Show `BT Gun Desktop`, session state, and active LAN endpoint `host:port`. |
| Primary area | Show QR code as the largest object on the screen, minimum 240x240, with expiry countdown visible within the same viewport. |
| Manual fallback | Always visible below or beside QR: endpoint, port, 6-digit code, and fingerprint suffix. This is secondary/debug, not hidden. |
| Primary action | `Start pairing` creates one short-lived pairing window. After start, label changes to `Restart pairing`. |
| Session states | `idle`, `pairing ready`, `android connected`, `authenticated`, `degraded`, `disconnected`, `expired`, `rate limited`. |
| Secret handling | Never show QR secret, HMAC proof, private key, or full token text in logs or diagnostic text. |

### Android Host Pairing Surface

| Region | Contract |
|--------|----------|
| Desktop link row | Replace Phase 2 placeholder with real state: no desktop, scanning, connecting, proving identity, connected, degraded, disconnected, trust problem. |
| Primary action | Show `Scan desktop QR` when no trusted desktop is selected. Show `Use trusted desktop` only after a desktop is remembered. |
| Manual fallback | `Enter manually` is visible next to scan action and opens fields for host/IP, port, and 6-digit code. |
| Trust confirmation | First successful pair stores desktop fingerprint. Reconnect with mismatched fingerprint must block and show `Desktop identity changed`. |
| Heartbeat | Show heartbeat age in seconds only for pairing/control channel. No packet loss, jitter, frame-rate, or visualizer latency in Phase 3. |
| Packet stream | Must remain inactive with Phase 4 placeholder copy. Do not make packet stream look connected in Phase 3. |

### Form And State Rules

| Case | Required Behavior |
|------|-------------------|
| QR expired | Stop connecting, show expiry error, keep `Rescan QR` and `Enter manually` available. |
| Endpoint unreachable | Show reachable next steps: rescan or edit manual endpoint/code. Do not start LAN discovery. |
| Wrong manual code | Show `Pairing code was rejected. Check the code and try again.` Apply rate-limit state after repeated failures. |
| Degraded heartbeat | Keep session visible, color warning amber, and show heartbeat age. |
| Disconnected | Preserve trusted desktop identity and show reconnect action. |
| Haptic command received | Show reserved/unsupported status only. Do not vibrate phone or show ack/fail execution copy. |
