## Phase Goal

**As a** user with the Android host app and desktop companion, **I want to** pair Android and desktop locally without manual IP entry, **so that** the devices establish a trusted authenticated session for later input streaming.

## Source Audit

| Source | Item | Coverage |

| RESEARCH | JSON control envelopes, heartbeat, diagnostics, profile metadata, haptic reservation | Covered by models, tests, and protocol docs |
| CONTEXT | D-15, D-16, D-17, D-18 | Covered by task actions below |

## Execution Boundary Note

This plan depends on the WSS envelope/server/client core from Plan 03-04. It must not change pairing proof behavior except where tests need existing proof-gated sessions. It must not add UDP input streaming, packet-loss metrics, visualizer latency, phone vibration, haptic payload fields, or haptic execution result semantics.
