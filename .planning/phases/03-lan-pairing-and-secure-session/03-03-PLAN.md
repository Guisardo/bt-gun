<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Android pairing proof to desktop verifier | Untrusted LAN client attempts to prove possession of one-time material. |
| Android trusted store to reconnect decision | Persisted identity controls whether a desktop is trusted. |
| Diagnostics/logs to user/developer | Security-sensitive material could leak into text. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | `TrustedDesktopStore.validateIdentity` | mitigate | Pin SPKI fingerprint and return `Mismatch` without overwrite on changed identity. |
| T-03-02 | Spoofing | manual code verification | mitigate | Enforce 6-digit code with TTL, attempt limit, and rate-limited state. |
| T-03-03 | Repudiation/Tampering | `PairingProof` | mitigate | Include nonces, session id, fingerprint, and one-time material in HMAC transcript; consume accepted sessions. |
| T-03-04 | Elevation of Privilege | `PairingSessionRegistry.verifyProof` | mitigate | Return no trusted session object until proof succeeds. |
| T-03-06 | Information Disclosure | diagnostics/logs | mitigate | `SecretRedactor` removes code, QR secret, proof, and private key material. |
</threat_model>
