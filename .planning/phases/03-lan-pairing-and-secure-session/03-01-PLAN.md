<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Desktop UI to LAN QR/manual payload | One-time material leaves the desktop display and may be photographed or typed. |
| Desktop key store to pairing payload | Public fingerprint may be shown; private key must stay local. |
| Package registry to build | New Maven dependencies enter the trusted build. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | `DesktopIdentityStore` | mitigate | Use SPKI SHA-256 fingerprint as the desktop identity shown in QR/manual UI; later plans pin it on Android. |
| T-03-02 | Tampering | `PairingSessionRegistry` manual code | mitigate | Generate a per-session 6-digit code with TTL and bind it to `sid`; rate limit enforcement lands in Plan 03-03. |
| T-03-06 | Information Disclosure | `PairingWindow`, diagnostics, logs | mitigate | `SecretRedactor` hides QR secret, manual code, proof, and private key; UI shows only fingerprint suffix. |
| T-03-SC | Tampering | Maven dependencies | mitigate | Blocking human package legitimacy checkpoint before Gradle dependency installation. |
</threat_model>
