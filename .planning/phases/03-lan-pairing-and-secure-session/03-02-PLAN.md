## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| QR/manual text to Android parser | Untrusted payload data enters Android from camera or typed fields. |
| Android trusted store to UI/session | Persisted desktop identity metadata influences trust decisions. |
| Android dashboard to user | Diagnostic text must not leak one-time material. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | `TrustedDesktopStore.validateIdentity` | mitigate | Store fingerprint as durable identity; later proof plan fails closed on mismatch. |
| T-03-05 | Denial of Service | `PairingPayload.parse` | mitigate | Reject malformed, oversized, expired, or unsupported-version payloads with typed errors. |
| T-03-06 | Information Disclosure | `DashboardState`, `TrustedDesktopStore` | mitigate | Store and render fingerprint suffix and state only; never store or show QR secret, manual code after entry, or proof. |
</threat_model>
