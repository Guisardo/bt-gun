<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ref archive -> static tools | Untrusted APK/XAPK bytes enter local reverse-engineering tools |
| static clue -> committed docs | Decompiled observations become repo evidence and can bias later implementation |
| ignored evidence -> committed manifest | Local raw/decompile paths are referenced without committing large or sensitive blobs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Denial of Service | `docs/refs` archive parsing | mitigate | Use `apktool`, `jadx`, and zip listing; do not execute vendor code. |
| T-01-02 | Information Disclosure | `.evidence/phase1` raw and decompile output | mitigate | Add `.gitignore` rules and commit only sanitized docs, manifests, and small fixtures per D-10. |
| T-01-03 | Tampering | Static clue verification status | mitigate | Validator full mode rejects verified status without clue, capture, and fixture linkage per D-06. |
| T-01-SC | Tampering | npm, pip, cargo, Gradle, or package installs | mitigate | Plan 01 performs no installs; any added package manager dependency must stop for blocking human legitimacy verification before use. |
</threat_model>
