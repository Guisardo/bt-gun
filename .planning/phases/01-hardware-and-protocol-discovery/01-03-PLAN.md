## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Android device -> repo evidence | Device logs and captures become committed summaries and manifests |
| physical gun -> diagnostic module | Unknown Bluetooth behavior crosses into diagnostic code and capture logs |
| Gradle metadata -> local build | Generated dependencies can execute downloaded build code if built |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Spoofing | Bluetooth scan result identity | mitigate | Record device name/address when safe, service ids, capture time, action context, and clue id; do not infer protocol from scan name alone. |
| T-01-08 | Information Disclosure | logcat, bugreport, HCI, and app-observed captures | mitigate | Store raw captures under ignored `.evidence/phase1/`; commit only sanitized manifest metadata per D-10. |
| T-01-09 | Denial of Service | Android permission or device unavailable state | mitigate | Diagnostic records denied permissions and unavailable hardware as explicit outcomes rather than hanging capture flow. |
| T-01-SC | Tampering | Gradle, npm, pip, cargo, or package installs | mitigate | Blocking human checkpoint reviews dependency coordinates and package legitimacy before build, install, or generated dependency acceptance. |
</threat_model>
