# Phase 08 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Boundary

Phase 8 is an Android-owned profile and mapping UI phase. The stale roadmap and requirement wording that says "desktop profiles" is superseded by `08-CONTEXT.md`.

Android stores, edits, validates, selects, deletes, and applies profiles. Desktop does not edit profiles; it only shows active Android profile metadata, revision, and mapped-stream diagnostics.

Out of scope for this UI contract: Phase 9 visualizer controls, latency dashboard, packet-loss dashboard, haptic test expansion, game-specific presets, import/export, direct desktop-to-gun Bluetooth, physical gun motor rumble, and new macOS virtual HID UI.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Android native Views for host UI; Swing for desktop companion |
| Icon library | none |
| Font | Android system sans on host; Swing system Dialog on desktop |

No `components.json`, Tailwind, React, Next.js, Vite, or web component system is present. The shadcn gate is not applicable.

Use the existing utilitarian dashboard style: native controls, selectable diagnostic text, compact rows, explicit labels, and no decorative cards. Do not introduce Compose, a web UI kit, or custom iconography in Phase 8.

Primary scan order is active Android profile state first, then profile actions, then validation/raw-debug status, then lower-priority diagnostics.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline label gaps, validation marker spacing |
| sm | 8px | Field vertical padding, button gaps, subsection gaps |
| md | 16px | Android root padding, row gaps, editor group padding |
| lg | 24px | Desktop BorderLayout gaps, major Android form breaks |
| xl | 32px | Desktop window outer border, profile editor top/bottom padding |
| 2xl | 48px | Android button/EditText minimum touch height |
| 3xl | 64px | Reserved page-level spacing; avoid unless a new full-screen editor needs separation |

Exceptions: Android tap targets and text inputs must be at least `48dp` high. Existing aim graph remains `220dp` high. Existing desktop QR remains `420px` square. These are stable fixed-format UI elements and must not be resized by dynamic labels.

## Typography

Use exactly these sizes and weights for new Phase 8 UI. Do not add more type sizes unless an existing platform widget imposes one.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14sp/14pt | 400 | 1.5 |
| Label | 13sp/13pt | 400 | 1.4 |
| Heading | 16sp/16pt | 700 | 1.2 |
| Display | 22sp/22pt | 700 | 1.2 |

Body copy is for dashboard rows, validation text, and diagnostic values. Label is for compact numeric values such as sensitivity, dead zone, smoothing lag, profile revision, and mapped-stream state. Heading is for profile editor section titles. Display is reserved for the existing `BT Gun Host` and `BT Gun Desktop` titles only.

Supporting existing colors: `#1F2933` for primary text and borders, `#4E5D6C` for secondary axes/diagnostic dividers, `#345BB2` for captured calibration marks, and `#B35000` for non-destructive warning/target indicators.

## Android Host Contract

### Dashboard Additions

Add profile rows to the existing Android dashboard near aim calibration and recenter state, before raw/debug panels:

| Row | Required Format |
|-----|-----------------|
| Active profile | `Active profile: {name} | id={profile_id} | rev={revision} | built_in={true/false}` |
| Profile mapping | `Profile mapping: mapped | sensitivity={value} | dead_zone={value} | smoothing={mode} | filter_lag<={ms}ms` |
| Recenter control | `Recenter control: hold {physical_button} for 2000ms` |
| Raw debug stream | `Raw debug stream: off` or `Raw debug stream: on | Android session controlled` |
| Profile error | `Profile error: none` or `Profile error: {validation_label}` |

Add a dashboard action labeled `Edit profiles`. It opens the Android profile management surface. Keep the existing `Start live session`, `Test local haptic`, Bluetooth gamepad actions, pairing actions, and debug provenance actions intact.

The raw debug toggle must live on Android. Desktop must not request raw extras in Phase 8.

### Profile List

Use a scrollable vertical Android native view. The first row is always `Default Visualizer` and must display `Built-in`, `Active` when selected, and `rev=1`. Built-in profile actions: `Use profile` and `Duplicate profile`; no rename or delete.

User-created profile rows display name, active state, revision, last updated timestamp if available, and actions: `Use profile`, `Edit profile`, `Duplicate profile`, `Delete profile`.

Empty state appears only when no user-created profiles exist, below the built-in row:

```text
No user profiles
Duplicate Default Visualizer to create an editable profile.
```

### Profile Editor

The editor must expose the full profile model in one flow. Do not hide provider-specific settings behind a future phase.

Required editor order:

1. Profile name text field.
2. Shared aim settings.
3. Provider override groups.
4. Button mapping.
5. Hold-to-recenter control.
6. Validation summary and `Save profile`.

Shared aim controls:

| Setting | Control | Default |
|---------|---------|---------|
| Sensitivity | Slider or numeric stepper with value label | `1.0` |
| Invert X | Switch/checkbox | off |
| Invert Y | Switch/checkbox | off |
| Dead zone | Slider or numeric stepper with value label | `0.03` |
| Smoothing | Segmented control or spinner: `Off`, `Low`, `Balanced`, `High`, `Adaptive` | `Low` |

Provider override groups must be visible in the main editor:

| Group | Copy | Default State |
|-------|------|---------------|
| Calibrated / fused rotation | Calibrated and fused rotation | Use shared settings |
| Gyro / raw aim | Gyro and raw aim | Use shared settings |
| Tilt fallback | Accelerometer and gravity tilt fallback | Use shared settings |

Each provider group has a `Use shared settings` switch/checkbox. When disabled, show the same five aim controls as shared settings within that group.

Adaptive smoothing status must be visible when live input exists:

```text
Smoothing: {mode} | filter lag <= {ms}ms
```

If the filter exceeds the allowed lag or the latency target headroom is too small, the live status must read:

```text
Smoothing: Low | adaptive fallback
```

### Button Mapping

Supported physical buttons: trigger, reload, X, Y, A, B.

Supported virtual outputs: trigger, reload, X, Y, A, B.

Use one row per physical button with a spinner/dropdown for the virtual output. Stick axes and aim axes must not be exposed as mapping choices. Do not allow button-to-axis, axis-to-button, or stick-axis remapping in Phase 8.

Default button mapping:

| Physical | Virtual |
|----------|---------|
| trigger | trigger |
| reload | reload |
| X | X |
| Y | Y |
| A | A |
| B | B |

### Hold-To-Recenter

Show a required single-choice control labeled `Hold-to-recenter button`. Choices: trigger, reload, X, Y, A, B.

Default value: reload.

Every saved profile must have exactly one physical hold-to-recenter button. Recenter is a physical gesture selection, not a virtual output mapping. Moving recenter away from reload must not break normal reload press/release output.

### Validation

Block `Save profile` and show inline errors for:

| Error Label | When |
|-------------|------|
| `Name required` | Profile name is blank |
| `Missing trigger output` | No physical button maps to virtual trigger |
| `Missing reload output` | No physical button maps to virtual reload |
| `Missing X output` | No physical button maps to virtual X |
| `Missing Y output` | No physical button maps to virtual Y |
| `Missing A output` | No physical button maps to virtual A |
| `Missing B output` | No physical button maps to virtual B |
| `Duplicate output` | More than one physical button maps to the same required virtual output |
| `Recenter required` | No hold-to-recenter physical button is selected |
| `Unsupported axis mapping` | Any UI/model attempt maps a button to an axis or an axis to a button |

Do not silently auto-repair invalid mappings.

## Desktop Companion Contract

Desktop remains a read-only status surface. Extend `PairingWindow` diagnostics only.

Required diagnostic rows:

| Row | Required Format |
|-----|-----------------|
| Active Android profile | `{name} | id={profile_id} | rev={revision}` |
| Profile source | `android` or `unknown` |
| Mapped stream | `{active/grace/stale/stopped} | mapped={true/false} | raw_debug={on/off}` |
| Last profile update | `none` or elapsed/status text without wall-clock precision requirements |

Allowed desktop controls remain `Start pairing`, `Restart pairing`, and the existing authenticated haptic smoke button. Do not add profile create, edit, rename, delete, sensitivity, dead-zone, smoothing, inversion, recenter, or raw-stream request controls to desktop.

Desktop copy must make Android authority explicit:

```text
Active Android profile
Mapped stream
```

Do not show Phase 9 visualizer data such as live button grid, aim reticle, latency chart, packet-loss chart, recenter history, or haptic test dashboard in this phase.

## State Contract

| State | Android UI | Desktop UI |
|-------|------------|------------|
| No profile store yet | Load `Default Visualizer`; show `Active profile: Default Visualizer` | Show `Active Android profile: unknown` until Android metadata arrives |
| Built-in active | Built-in row selected; edit disabled; duplicate allowed | Show name/id/revision from Android metadata |
| User profile active | User row selected; edit/delete allowed | Show name/id/revision from Android metadata |
| Validation blocked | Save disabled; top summary `Save blocked`; inline errors visible | No desktop error unless Android publishes invalid/unset metadata |
| Raw debug off | Dashboard says `Raw debug stream: off`; product mapped stream only | `raw_debug=off` |
| Raw debug on | Dashboard says `Raw debug stream: on | Android session controlled` | `raw_debug=on`; still no request control |
| Stream stale | Android packet row uses existing lifecycle label | Desktop `Mapped stream: stale ...` |

## Verification And Evidence Contract

Phase 8 UI implementation must be verified on the connected USB Android device with device screenshots. This is evidence for Android profile UI correctness only; it must not expand Phase 8 into the Phase 9 visualizer.

Required screenshot evidence states:

| Capture | Required UI Evidence |
|---------|----------------------|
| `phase8-dashboard-profile-rows` | Android dashboard shows active profile, mapped profile settings, recenter control, raw debug state |
| `phase8-profile-list-default` | Profile list shows immutable `Default Visualizer`, active badge, duplicate action, and no delete action |
| `phase8-profile-editor-provider-overrides` | Editor shows shared aim settings plus all provider override groups in the main flow |
| `phase8-validation-blocked-save` | Invalid profile blocks save and shows validation summary plus inline error |
| `phase8-raw-debug-toggle` | Android raw debug toggle changes Android session state; desktop has no request control |

Storage rule for screenshots:

- Store raw screenshots only under an ignored evidence directory such as `.evidence/phase8/android-profile-ui/<run-id>/` after adding that path to `.gitignore`, or under `/private/tmp`.
- Before phase closeout, remove stale or unused screenshots from the Phase 8 evidence directory. Keep only the current evidence set needed for verification.
- Do not commit raw screenshots, screenshot paths that expose sensitive device names, Bluetooth addresses, serials, account names, pairing material, stream secrets, proof values, private keys, raw packet dumps, or raw logs.
- Commit only sanitized manifest rows or summary notes with stable capture ids and pass/fail status.

Recommended automated checks before human/device sign-off:

```bash
./gradlew :android-host:app:test
./gradlew :desktop-companion:test
rg -n "Desktop profile editor|Edit desktop profile|Request raw stream" android-host desktop-companion .planning docs
```

Recommended device evidence commands:

```bash
adb devices
adb exec-out screencap -p
```

The executor must delete stale unused Phase 8 screenshot files after capturing the final evidence set.
