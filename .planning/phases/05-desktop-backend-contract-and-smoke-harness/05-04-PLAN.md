## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Smoke fixture bytes -> backend publish | Fake input must still cross receiver authentication/replay boundary before publish. |
| Smoke XML -> phase evidence | XML must be parseable, platform-distinguishable, and free of secret material. |
| Platform stub command -> production driver expectations | Stub commands must not claim OS-visible HID support. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-05-11 | Information Disclosure | `JunitSmokeXml` | mitigate | XML includes case names/results only; no QR/manual/proof/stream key/HMAC/private key text is written. |
| T-05-12 | Repudiation | Platform smoke commands | mitigate | Distinct macOS and Windows XML paths and suite names make platform runs auditable per D-07/D-08. |

| T-05-SC | Tampering | Package installs | accept | No package-manager install is planned; XML writer is local code, no dependency. |
</threat_model>
