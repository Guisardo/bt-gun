| BLE `fff3` bytes -> product gun events | Untrusted device payload crosses parser whitelist boundary. |
| Parser -> debug provenance | Raw bytes and evidence ids become available only as optional debug metadata. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-05 | Tampering | `IpegaPacketParser.parseFff3` | high | mitigate | Whitelist Phase 1 fixture payloads; unknown bytes emit `UnknownBlePayload` only. |
| T-02-06 | Information Disclosure | `Provenance` | medium | mitigate | Preserve provenance as optional debug metadata; UI product mode hides it in Plan 02-06. |
| T-02-07 | Repudiation | Fixture parser tests | medium | mitigate | Tests assert `clueId`, `captureId`, raw ASCII/hex, and characteristic linkage for known controls. |
| T-02-08 | Spoofing | Connection/status event mapping | medium | mitigate | Parser does not mark arbitrary payloads as connection state; only adapter/service can emit connection status. |
| T-02-SC | Tampering | Package installs | high | mitigate | No new package installs; use committed fixtures and JVM tests only. |
</threat_model>
