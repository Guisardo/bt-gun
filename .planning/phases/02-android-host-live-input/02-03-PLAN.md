<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Android runtime -> BLE APIs | Permission-gated app code invokes privileged scan/connect/GATT operations. |
| Physical gun BLE -> adapter/parser | Untrusted `fff3` notifications enter host app. |
| Service lifecycle -> background execution | Active BLE session continues only as visible foreground service. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-09 | Elevation of Privilege | `HostSessionService` BLE start | high | mitigate | Refuse scan/connect when `PermissionGateState` is blocked; catch `SecurityException` into visible last error. |
| T-02-10 | Denial of Service | `ReconnectPolicy` | high | mitigate | Bound reconnect attempts/backoff while active; no reconnect after explicit stop; show `Reconnecting...` plus last error. |
| T-02-11 | Tampering | `IpegaBleGunAdapter` | high | mitigate | Accept only `ARGunGame`/`fff0`; parser whitelist handles `fff3` values; unknown devices ignored. |
| T-02-12 | Elevation of Privilege | foreground service policy | high | mitigate | Declare connected-device foreground service and show exact active notification before long-running BLE work. |
| T-02-13 | Scope/Safety | `fff5`/haptics | high | mitigate | Do not write `fff5` or implement desktop-origin haptic commands in Phase 2. |
| T-02-SC | Tampering | Package installs | high | mitigate | No BLE helper packages or mocking libraries; platform APIs and pure state tests only. |
</threat_model>
