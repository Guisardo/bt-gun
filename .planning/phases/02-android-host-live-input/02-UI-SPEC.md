## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline status text gaps, divider offsets |
| sm | 8px | Compact row gaps, chip padding, debug metadata spacing |
| md | 16px | Default screen padding, card padding, form field spacing |
| lg | 24px | Section padding, distance between dashboard groups |
| xl | 32px | Major vertical breaks between permission gate and dashboard content |
| 2xl | 48px | Full-width primary actions, minimum touch targets, top/bottom safe-area offsets |
| 3xl | 64px | Reserved for large-screen/tablet split spacing only |

Exceptions:

| Exception | Value | Reason |
|-----------|-------|--------|
| Minimum touch target | 48px x 48px | Android operational use while holding the physical gun |
| Preview aim pad | min 192px x 192px; preferred 240px x 240px; max 320px x 320px | Stable square target area for motion preview and recenter feedback |
| Status chip height | 32px | Dense dashboard scanning without shrinking touchable controls |
| Expandable debug header | 48px height | Full-row tap target for raw BLE/provenance panels |

Spacing contract:

- Use a single vertical scroll surface on phones. No nested scrolling inside dashboard cards.
- Use 16px horizontal screen padding on phones and 24px on tablet/wide layouts.
- Cards use 8px radius max, 1px border, and 16px internal padding. Do not put cards inside cards.
- Fixed-format elements such as aim pad, status chips, and icon buttons must have stable dimensions so live values do not shift layout.

## Typography

Use exactly these four type sizes and exactly these two weights.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px | 600 | 1.3 |
| Body | 14px | 400 | 1.5 |
| Control | 16px | 600 | 1.35 |
| Heading | 22px | 600 | 1.2 |

Typography contract:

- Use 14px body text for dashboard values, event descriptions, permission explanations, and debug rows.
- Use 12px semibold labels for field labels, stream names, sequence labels, provider metadata, and placeholder labels.
- Use 16px semibold for primary buttons, section actions, and active status values.
- Use 22px semibold only for the screen title or permission-gate title. Do not use hero-scale type.
- Use tabular numerals where available for sequence numbers, elapsed time, axes, and timestamps.
- Do not scale font size with viewport width. Letter spacing is 0.

- Active motion provider label when provider is valid.
- Preview aim reticle and recenter baseline marker.
- Recenter completed feedback.
- Phone vibration local test success.
- Selected segment in the Product Events / Debug Provenance toggle.

Color contract:

- Default text is #1F2933. Secondary text is #52606D. Borders are #D9E2EC.
- Inactive Phase 3 or Phase 4 placeholders use #E5E7EB surface, #6B7280 text, and no accent.
- Errors must use destructive color plus plain text. Do not rely on color alone.
- Connecting/reconnecting states use neutral secondary styling with an inline progress indicator, not accent success styling.
- Avoid decorative gradients, bokeh, dark dashboard chrome, and one-hue theme treatments.

## Screen Contract

### Permission Gate

- Show only when one or more required permissions are missing or blocked.
- Present a compact checklist for Bluetooth scan/connect, nearby devices, location needed for scan compatibility, sensors, LAN/network state, and vibration capability.
- Primary action is `Grant permissions`.
- If a permission is permanently denied, show: `Open Android settings to enable this permission.`
- Do not show raw BLE details on the permission gate unless the user opens the Permission state debug panel.

### Main Dashboard

The first screen after permissions is the operational dashboard. Order sections top to bottom:

1. Top app bar: `BT Gun Host`, foreground-service indicator, overflow/debug access.
2. Session status strip: gun connection state, reconnect state, current error line.
3. Primary action row: `Start live session` before running; `Stop session` while running; `Test phone vibration` as secondary action.
4. Live input group: last gun event, reload state, trigger state, stick direction, X/Y/A/B button state.
5. Motion group: active provider, provider capability flags, preview aim pad, current zero/baseline timestamp.
6. Recenter group: reload-hold countdown, recenter fired state, last recenter timestamp.
7. Phase placeholders: Desktop link and Packet stream as inactive surfaces with explicit pending text.
8. Haptic group: phone vibrator capability and last local phone-vibration test result only.
9. Debug panels: BLE provenance, Permission state, GATT status; collapsed by default.

### Live Values

- Update live values in place. Do not append an endless log to the main dashboard.
- Last event shows event name, down/up state, stream sequence, and elapsed timestamp.
- Product mode hides raw bytes, characteristic UUIDs, clue ids, capture ids, and confidence values.
- Debug provenance mode reveals raw ASCII/hex, BLE service/characteristic, `clue_id`, `capture_id`, semantic confidence, GATT status, and permission details.
- Desktop link and packet stream must not imply a working LAN path in Phase 2.

### Preview Aim

- Use a stable square pad with centered crosshair, current aim dot, and recenter baseline marker.
- Show derived X/Y numeric values below the pad using tabular numerals.
- Label Android-side aim as preview/calibration only. Do not expose desktop profile tuning in Phase 2.
- If motion provider is unavailable, keep the pad visible but disabled and show `Motion unavailable`.

## Interaction Contract

### Live Session

- `Start live session` runs permission checks, starts the foreground service, scans for `ARGunGame` advertising service `fff0`, connects, and subscribes to live input.
- Auto-connect is the normal path after permissions pass. Do not show a manual device picker as the default product flow.
- Auto-reconnect runs while the active session is running. Show `Reconnecting...` and last error rather than silently retrying.
- `Stop session` stops scan/connect work and the foreground service. It is non-destructive and needs no confirmation.

### Reload Hold Recenter

- On reload press, immediately emit and display `reload down`.
- During hold, show a visible two-second progress state after the first 500ms.
- At two seconds, emit and display `recenter emitted` without consuming reload state.
- On release, emit and display `reload up`.
- If reload releases before two seconds, remove countdown and do not emit recenter.

### Motion Provider

- Provider selection order is game rotation vector, rotation vector, gyro plus gravity/accelerometer, then gravity/accelerometer tilt fallback.
- Display provider name, capability flags, and capture timestamp source.
- If provider changes mid-session, show the new provider in the motion group and add one product event line for the provider change.

### Haptics

- Phase 2 haptic status means Android phone vibrator capability and last local test only.
- `Test phone vibration` runs a local phone vibration test and shows last result as success, failed, or unavailable.
- Do not show desktop-origin haptic controls or ack/fail states until Phase 4.

### Debug Panels

- Debug panels are collapsed by default.
- Expanding a debug panel must not move the primary action row off-screen on typical phones; place panels after product dashboard content.
- Debug provenance toggle changes detail level, not the underlying event stream.
- Raw values are selectable/copyable when visible.

## Accessibility and Device Constraints

- All touch targets are at least 48px x 48px.
- The dashboard must work one-handed in portrait on a normal Android phone while the user holds the physical gun.
- All status colors must also have text labels.
- Aim preview must remain understandable without color by using crosshair, dot, baseline marker, and numeric axes.
- Text must wrap inside cards and buttons; never truncate permission or error instructions without an accessible expanded value.
- Foreground-service notification copy should state active behavior: `BT Gun Host running - live input active`.
- Keep wake/active-session behavior explicit in UI; do not hide that the foreground service is running.
