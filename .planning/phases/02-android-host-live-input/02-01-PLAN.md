<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User -> Android runtime permissions | User grants Bluetooth/location/network/vibration capabilities before scan/session work. |
| Android app -> future LAN/desktop surfaces | Phase 2 shows inactive surfaces only; no network protocol or desktop haptic command is accepted. |
| Product UI -> debug provenance | Raw BLE/provenance fields cross from internal debug model to user-visible panels only when debug mode is enabled. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Elevation of Privilege | `PermissionGate`, manifest | high | mitigate | Gate BLE scan/connect on runtime permission state; tests require blocked state before session start. |
| T-02-02 | Information Disclosure | `Provenance`, debug panels | medium | mitigate | Product mode hides raw payload/address/clue/capture fields; debug provenance explicit and collapsed by default. |
| T-02-03 | Tampering | `DashboardState` future desktop surfaces | high | mitigate | Desktop link and packet stream are inactive Phase 3/4 surfaces with exact pending copy and no send/connect action. |
| T-02-04 | Tampering | Package/dependency surface | high | mitigate | No new dependencies; any new Gradle coordinate requires blocking human dependency checkpoint before build. |
| T-02-SC | Tampering | npm/pip/cargo/Gradle installs | high | mitigate | Reuse approved AGP/Kotlin plugin versions only; no package-manager install task in this plan. |
</threat_model>
