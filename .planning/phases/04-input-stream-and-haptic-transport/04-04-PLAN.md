## Phase Goal

**As a** player using the Android gun host and desktop companion, **I want to** stream high-rate input and return phone haptics over the trusted LAN session, **so that** the desktop receives current controls and the phone only vibrates for valid fresh commands.

## Source Audit

| Source | Item | Coverage |
|--------|------|----------|
| GOAL | Haptic commands travel back to Android phone with acknowledgements | This plan implements reliable command/result transport and Android phone pulse execution |
| REQ | ANDR-07 | Covered by Android receiving a desktop-origin command and vibrating phone |
| REQ | TRAN-07 | Covered by desktop command body fields |
| REQ | TRAN-08 | Covered by Android result envelope statuses |
| RESEARCH | Use WSS for haptics, relative TTL, extend `PhoneHaptics`, keep no new deps | Covered by command/result models and executor |
| CONTEXT | D-06, D-07, D-08, D-09, D-10, D-11, D-14 | Covered by control migration, latest-valid-wins, ack/fail timing, statuses, expiry, and cancellation policy |
| CONTEXT deferred | Full haptic pattern playback and physical gun motor rumble | Not implemented; non-null pattern returns `unsupported` without vibration |

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Desktop control command to Android haptic executor | Remote desktop request can trigger local phone hardware. |
| Control body JSON to command model | Malformed or oversized control body must not execute. |
| Android haptic executor to platform vibrator | Platform failures and permission errors must become explicit result statuses. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-14 | Spoofing/Elevation of Privilege | `DesktopControlClient` haptic handling | mitigate | Accept haptic commands only through authenticated WSS with matching trusted session id. |
| T-04-15 | Tampering | haptic command body | mitigate | Validate command id, strength, duration, TTL, and pattern before any vibration call. |
| T-04-16 | Denial of Service | stale haptics | mitigate | Enforce TTL and latest-valid-wins cancellation; expired commands do not vibrate. |
| T-04-17 | Repudiation | haptic result | mitigate | Return `haptic_result` with command id, status, detail, and observed elapsed nanos after start attempt. |
| T-04-18 | Information Disclosure | haptic/control logs | mitigate | Do not log QR secret, manual code, proof, stream key, or HMAC key material in result details. |
</threat_model>
