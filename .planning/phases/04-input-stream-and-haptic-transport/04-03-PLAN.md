<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Trusted WSS session to UDP config | Desktop creates the stream key/id only after auth. |
| LAN UDP to desktop receiver | Untrusted datagrams can be spoofed, replayed, malformed, or delayed. |
| Receiver state to later backend | Raw input state must not pretend to be mapped joystick output. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | Spoofing/Tampering | `UdpInputReceiver` | mitigate | Verify HMAC-SHA256 with per-session key before applying any frame payload. |
| T-04-10 | Tampering | `InputReplayGuard` | mitigate | Reject wrong-session, duplicate-sequence, old-sequence, late edge, bad-MAC, and age-expired frames per D-12. |
| T-04-11 | Denial of Service | UDP frame parser | mitigate | Fixed 120-byte frame length, magic/version/type checks, and no dynamic allocations from untrusted length fields. |
| T-04-12 | Information Disclosure | stream config and diagnostics | mitigate | Do not log or display stream key, QR secret, manual code, proof material, or HMAC key bytes. |
| T-04-13 | Tampering | timeout recovery | mitigate | On timeout clear buttons only, retain last aim as stale, and surface stale state per D-13. |
</threat_model>
