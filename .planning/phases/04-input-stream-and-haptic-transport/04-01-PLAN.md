## Phase Goal

**As a** player using the Android gun host and desktop companion, **I want to** stream high-rate input and return phone haptics over the trusted LAN session, **so that** the desktop receives current controls and the phone only vibrates for valid fresh commands.

## Source Audit

| Source | Item | Coverage |
|--------|------|----------|
| GOAL | Desktop safely receives high-rate Android input | This plan creates the authenticated binary frame contract consumed by sender and receiver plans |
| REQ | TRAN-05 | Covered by fixed fields for session id, sequence, timestamps, buttons, axes, motion payload, and provider/capability flags |
| REQ | DESK-01 | Partially covered by desktop codec HMAC validation and parse contract before Plan 04-03 receiver application |
| REQ | PERF-03 | Partially covered by bad-MAC/wrong-session/fixture rejection before Plan 04-03 replay guard |
| RESEARCH | HMAC-only UDP, fixed big-endian layout, mirrored codecs plus golden fixtures | Covered with no new dependencies |
| CONTEXT | D-01, D-02, D-03, D-04, D-05, D-12 | Covered explicitly in codec fields, debug decoder, and rejection tests |
| CONTEXT deferred | Full haptic pattern playback, physical gun motor rumble, virtual joystick, profile mapping, visualizer metrics | Not planned in this file |

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LAN UDP datagram to codec | Untrusted bytes reach parser/authentication code. |
| Trusted WSS config to UDP sender/receiver | Stream id/key material crosses from reliable control into UDP transport. |
| Debug decoder to logs/docs | Diagnostic output can leak secrets if not redacted. |
| Build tooling to validation | Broken local Gradle startup can produce false confidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing/Tampering | `UdpInputFrameCodec` | mitigate | HMAC-SHA256 authenticates fixed binary frames before any payload apply; malformed magic/version/type/length fail closed. |
| T-04-02 | Tampering | replay/stale frame setup | mitigate | Include stream id, sequence, capture/send timestamps, and age-limit config so Plan 04-03 can reject wrong-session, duplicate, old, and age-expired frames per D-12. |
| T-04-03 | Information Disclosure | debug decoder/docs/tests | mitigate | Debug output may show frame type, sequence, provider, and rejection reason, but never stream key, QR secret, manual code, proof, or HMAC key material. |
| T-04-04 | Denial of Service | UDP binary parser | mitigate | Enforce fixed 120-byte frame length, magic/version/type checks, and no variable-size payloads. |
| T-04-SC | Tampering | dependency supply chain | mitigate | No new external packages are allowed; if an executor attempts one, stop for a package legitimacy gate before editing builds. |
</threat_model>
