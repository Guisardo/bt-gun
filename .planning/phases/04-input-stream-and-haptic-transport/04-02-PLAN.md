## Phase Goal

**As a** player using the Android gun host and desktop companion, **I want to** stream high-rate input and return phone haptics over the trusted LAN session, **so that** the desktop receives current controls and the phone only vibrates for valid fresh commands.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WSS control config to UDP sender | Trusted config material enables UDP; accepting it too early would bypass auth. |
| Android live state to LAN datagrams | Gun/motion state leaves device over local network. |
| Foreground service lifecycle to sockets | Sockets must not outlive visible active session and trusted stream policy. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Elevation of Privilege | `DesktopControlClient` input config | mitigate | Accept `input_stream_config` only after `session_ready` and matching trusted session id. |
| T-04-06 | Tampering | `AndroidUdpInputSender` datagrams | mitigate | HMAC every frame with per-session key from trusted control config. |
| T-04-07 | Information Disclosure | Android logs/debug | mitigate | Never log `hmacSha256KeyBase64Url`, QR secrets, manual codes, proof values, or raw HMAC key bytes. |
| T-04-08 | Denial of Service | sender lifecycle | mitigate | Stop sender on service stop/session change and enforce grace timeout after control disconnect per D-15. |
| T-04-SC | Tampering | dependency supply chain | mitigate | No new external packages; use platform UDP socket APIs and existing dependencies only. |
</threat_model>
