<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Physical Android/gun/LAN to committed evidence | Raw device/network observations become sanitized planning artifacts. |
| Trusted reconnect to old UDP frames | Old frames may still exist after a fresh stream config and must not be counted as valid input. |
| Desktop haptic command to Android vibrator | Human must distinguish valid fresh pulses from expired/cancelled/stale commands. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-24 | Information Disclosure | physical smoke evidence | mitigate | Store raw logs/screenshots only under ignored `.evidence/phase4/input-haptic-transport/`; commit only sanitized notes and stable capture ids. |
| T-04-25 | Tampering | manual evidence status | mitigate | Require expected and observed fields per capture id; failed/blocked rows must preserve reason instead of being treated as pass. |
| T-04-26 | Spoofing/Tampering | old UDP replay evidence | mitigate | Reconnect checks require fresh trusted config and old-frame rejection before input applies when replay/resend is available. |
| T-04-27 | Repudiation | haptic command result | mitigate | Record command class, expected status, observed status, and whether physical phone pulse occurred for valid, expired, session-change, and short-disconnect cases. |
</threat_model>
