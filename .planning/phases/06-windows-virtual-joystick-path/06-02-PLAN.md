## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User mode -> kernel IOCTL | Companion/helper bytes enter KMDF driver. |
| Windows HID stack -> driver output callback | OS-origin output reports enter driver queue. |
| Driver artifacts -> Windows target | Test-signed package later loads on `192.168.1.100`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Tampering | `IOCTL_BTGVJOY_SUBMIT_INPUT` | mitigate | Require `FILE_WRITE_DATA`, `METHOD_BUFFERED`, exact size, version, report ID, and bounded report bytes before VHF submit. |
| T-06-05 | DoS | VHF output queue | mitigate | Bound queued output reports and status counters; drop or report overflow without kernel allocation growth. |
| T-06-06 | Elevation of Privilege | KMDF driver | mitigate | Keep driver as HID/report bridge only; no LAN, crypto, profile, file parsing, or Android session state per D-12. |
| T-06-07 | Information Disclosure | Helper tools/logs | mitigate | Print no LAN/session secrets, pairing proofs, private keys, raw QR material, or device identifiers. |
| T-06-SC | Tampering | Package installs | accept | No package-manager install in this plan; WDK build happens in later CI plan per D-04 and D-05. |
</threat_model>
