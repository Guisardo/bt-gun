# Quick Task: Android Host Mobile UI Design Refresh

## Objective

Improve the Android host dashboard and profile editor using current Android and Material mobile design guidance while preserving Phase 8 profile behavior and native Android Views.

## Scope

- Update `android-host/app/src/main/java/com/btgun/host/MainActivity.kt`.
- Keep the implementation dependency-free: no Compose or Material Components migration.
- Verify with Android unit tests and USB-device screenshots.
- Keep screenshots under ignored `.evidence/phase8/android-profile-ui/`.

## Tasks

1. Apply modern native-Views styling.
   - Add section hierarchy, app header copy, readable row surfaces, and styled primary/tonal/danger buttons.
   - Add system bar inset handling and stable scroll padding for the target Android device.

2. Improve profile-management ergonomics.
   - Give profile editor controls explicit labels.
   - Keep controls at least 48dp high.
   - Stop periodic dashboard refresh from resetting profile-editor scroll while the profile surface is open.

3. Verify and capture evidence.
   - Run Android unit tests.
   - Install on the connected USB Android device.
   - Capture current dashboard/profile screenshots and delete stale or temporary screenshots locally and on device.

## Acceptance

- Android dashboard is easier to scan: permissions, session, profile, Bluetooth, live aim, diagnostics, pairing, and debug areas are visually separated.
- Profile list and editor keep all Phase 8 labels/actions and do not add desktop profile authority.
- User profile editor scrolls normally during editing.
- Unit tests pass and final evidence screenshots are retained only under the ignored Phase 8 evidence directory.
